# Mouse Jiggler

HID-приложение: переключает Flipper в режим USB-мыши и периодически делает крошечное движение, чтобы ПК не уходил в сон / не показывал «Away» статус в Teams/Slack.

## Что делает

При нажатии **OK**:
1. Переключает USB Flipper'а в режим HID-мыши
2. Каждые **N секунд** делает движение `+2,+2` → пауза 60мс → `-2,-2` (диагональный нюдж туда-сюда, в сумме ноль смещения)
3. Считает количество движений и время до следующего

## Управление

| Кнопка | Действие |
|--------|----------|
| OK | Включить / выключить jiggler |
| ↑ | Увеличить интервал (15s → 30s → 1m → 2m → 4m) |
| ↓ | Уменьшить интервал |
| Back | Выход (восстановит исходный USB-режим) |

## Совместимость

**Работает на Windows / macOS / Linux** без установки дополнительного софта — это обычная USB-мышь по HID-стандарту, любая ОС подхватит сразу.

| ОС | Особенности |
|----|-------------|
| Windows 10/11 | Работает «из коробки». Teams/Slack/Discord не покажут «Away» |
| macOS | Работает «из коробки». Может потребоваться разрешить «USB Input Device» в System Settings → Privacy → Accessibility если запрашивает |
| Linux | Работает «из коробки» (любой DE) |

## Антидетект

Движение `+2,+2 / -2,-2` подобрано чтобы:
- Не сбивать прицеливание/курсор пользователя (2 пикселя незаметно)
- Выглядеть как живое HID-событие (не одинаковое смещение)
- Иметь нулевую сумму (курсор возвращается на место)

Большинство «контролёров активности» (Teams, Slack, Microsoft Activity Insights, Hubstaff обычный режим) **видят это как реальную активность пользователя**. Более продвинутые системы (например DLP с поведенческим анализом) теоретически могут распознать паттерн — но это редкость.

## Сборка

```bash
cd mouse_jiggler
ufbt
```

Категория: `apps/USB/`.

## Юридическая оговорка

Используй на **своих** машинах. На корпоративном ноуте с явной политикой запрета «activity simulation tools» — это нарушение трудового договора, могут уволить. Серьёзно, не майся ерундой на работе.
