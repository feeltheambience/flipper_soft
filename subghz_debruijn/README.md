# SubGHz De Bruijn Brute

**Инструмент авторизованного пентеста.** Быстрый перебор fixed-code (CAME / NICE / Princeton / Holtek) через **De Bruijn-последовательность** — на порядки быстрее наивного покадрового брутфорса.

> ⚠️ Только для систем, которые ты владеешь или авторизован тестировать. Передача на чужие ворота/шлагбаумы — незаконна.

## Почему это быстро

Наивный bruteforce (как встроенный в прошивку) шлёт каждый код отдельным кадром: преамбула + код + пауза, ещё и ×3-5 повторов. De Bruijn — это **одна непрерывная битовая строка**, в которой каждая n-битная комбинация встречается ровно раз как скользящее окно. Нет преамбул, пауз, повторов.

```
CAME 12-bit (4096 кодов):
  Наивный:    4096 × 3 повтора × (~20мс) ≈ 5 минут
  De Bruijn:  (4096+11) бит × 960мкс      ≈ 4 секунды
```

**~75x быстрее.** (Каждый CAME-бит = 3·te = 960мкс PWM, поэтому не 1.5с, а ~4с — но всё равно секунды против минут.)

## Где работает / не работает

✅ **Работает** на приёмниках со скользящим окном / сдвиговым регистром — большинство дешёвых CAME/NICE/Princeton/Holtek клонов (ворота, шлагбаумы, радиорозетки до ~2010). Это классическая атака (OpenSesame Сэми Камкара).

❌ **НЕ помогает** против:
- **Rolling-code** (KeeLoq, Nice FloR-S, CAME Atomo, нормальные сигналки) — там код одноразовый, перебор бесполезен в принципе
- Приёмников, **строго требующих guard-паузу перед каждым кодом** — для них De Bruijn вырождается; нужно тестировать на конкретной цели

## Параметры

| Поле | Значения | Заметка |
|------|----------|---------|
| **Freq** | 433.92 / 868.35 МГц | Частота цели (узнай через Read/Frequency Analyzer) |
| **Bits** | 8 / 10 / **12** | Битность кода. CAME/NICE = 12 |
| **Repeats** | 1 / 2 / 3 | Проходов всей последовательности (для надёжности) |

Кодировка зашита под CAME/NICE: te = 320 мкс, бит `1` = HIGH te + LOW 2·te, бит `0` = HIGH 2·te + LOW te.

## Почему нет 24-bit

24-битный De Bruijn = 2²⁴ = 16.7М бит. Это:
- **Не влезает в RAM** Flipper (нужно ~16 МБ, есть 256 КБ)
- **Airtime ~1.6 часа** даже идеально — физический предел передачи

Поэтому приложение ограничено ≤12 бит (где атака реально быстрая). Если у твоей цели реально задействованы только младшие 8-12 бит 24-битного Princeton (частый случай с dip-переключателями) — ставь Bits=12.

## Как пользоваться

1. Определи частоту и протокол цели: на Flipper'е `Sub-GHz → Read`, нажми пульт, посмотри protocol/freq
2. Если это CAME/NICE 12-bit → открой это приложение, выставь Freq + Bits=12
3. Наведи Flipper на приёмник, OK → за ~4-8 сек прогонит все 4096 кодов
4. Если сработало (ворота дрогнули) — фиксируй в отчёт: «fixed-code, уязвим к De Bruijn brute»
