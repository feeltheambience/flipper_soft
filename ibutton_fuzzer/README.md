# iButton Sequential Fuzzer

**Инструмент авторизованного физического пентеста.** Демонстрирует уязвимость систем контроля доступа, выдающих **последовательные ID** (частая находка в домофонах / СКУД).

> ⚠️ **Только для систем, которые ты владеешь или авторизован тестировать.** Применение против чужого контроля доступа — незаконно (в РФ — ст. 272 УК и др.). Логи на SD служат audit-trail для отчёта по легальному заказу.

## Почему это НЕ «брутфорс с нуля»

Полное пространство iButton — **2⁴⁸ ≈ 2.8×10¹⁴** комбинаций. Ридеру нужно ~0.3-0.5 сек на попытку → перебор всего займёт тысячелетия. **Это физически невозможно.**

Реальная техника, которую реализует это приложение:
1. Ты легально считал **один валидный ключ** (например, свой ключ от подъезда)
2. Если система выдавала ключи **последовательно** (UID жильцов идут подряд), то соседние ID `база ± N` — тоже валидные
3. Перебор узкого диапазона вокруг базы за минуты находит чужие валидные ключи → доказательство уязвимости

Если система выдаёт **случайные** UID — этот метод ничего не найдёт (и это правильный результат: значит, СКУД устойчива).

## Как пользоваться

### 1. Считай базовый ключ
На Flipper'е: `iButton → Read` → приложи валидный ключ → `Save`. Файл ляжет в `/ext/ibutton/*.ibtn`.

### 2. Запусти фаззер
`Apps → iButton → iButton Seq Fuzzer`
1. **Выбери базовый ключ** из списка
2. Настрой параметры:
   - **Range** — насколько далеко перебирать (±10 … ±1000)
   - **Hold** — сколько держать каждый ID (150-1000 мс; больше = надёжнее читается ридером)
   - **Dir** — `up only +` (только вверх) или `both ±` (в обе стороны)
3. **OK** — старт. Прикладывай Flipper к ридеру/замку во время перебора

### 3. Смотри лог
Каждый эмулированный UID пишется в `/ext/apps_data/ibutton_fuzzer/fuzz_log.txt` с индексом и таймстампом — для отчёта.

## Поддержка протоколов

Работает с тем протоколом, в котором сохранён базовый ключ (DS1990/Dallas, Cyfral, Metakom — что Flipper прочитал). CRC пересчитывается автоматически при каждом инкременте.

## RFID 125 кГц и Sub-GHz — тот же принцип

### RFID (EM4100 и т.п.)
- **Полный перебор 2⁴⁰ — тоже невозможен** (та же причина: ридер медленный)
- Та же техника sequential-fuzz реализуется через `lfrfid_worker` API (эмуляция ID по аналогии с iButton)
- В Unleashed **уже есть `LF RFID Fuzzer`** — перебор по базе дефолтных ключей + инкремент. Для большинства задач его достаточно
- Если нужен наш вариант с audit-логом для RFID — скажи, добавлю режим (API уже изучен)

### Sub-GHz — здесь brute-force РЕАЛЬНО работает
- Для **fixed-code** (CAME 12bit=4096, NICE, Princeton, Holtek) перебор всего keyspace осмыслен — секунды/минуты через **De Bruijn последовательность**
- В Unleashed **уже встроен `SubGHz Bruteforcer`** — используй его
- **Rolling-code** (KeeLoq и пр.) — иммунны, перебор бесполезен
- Своя реализация требует полного subghz TX-пайплайна (тот же, что отложен в life_scenes) — добьём при необходимости с тестом на твоём железе
