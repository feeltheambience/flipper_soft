# Remote Hub

Быстрый запуск избранных Sub-GHz сигналов из одного плоского списка. Вместо `Apps → Sub-GHz → Saved → длинное_имя_папки → файл` — открыл Remote Hub, стрелка, OK.

## Что делает

- Сканирует папку **`/ext/subghz/remote_hub/`** на SD-карте на наличие `.sub` файлов
- Показывает список с прокруткой (до 32 файлов)
- По нажатию **OK** запускает встроенное Sub-GHz приложение в режиме отправки выбранного файла (там жмёшь и держишь OK для передачи сигнала)

Папка `/ext/subghz/remote_hub/` создаётся автоматически при первом запуске, если её нет.

## Управление

| Кнопка | Действие |
|--------|----------|
| ↑ / ↓ | Выбор файла в списке |
| OK | Открыть выбранный сигнал в Sub-GHz (или пересканировать папку, если она пуста) |
| Back | Выход |

## Как добавить свои сигналы

1. Подключи Flipper по USB и открой **qFlipper**
2. Зайди в **SD Card → subghz**
3. Скопируй (или drag-n-drop) свои `.sub` файлы в папку **`remote_hub/`**
   - Имена файлов = подписи в списке (без расширения)
   - Поддерживаются любые `.sub` файлы — RAW и декодированные (Princeton, CAME, Nice, KeeLoq и т.д.)
4. Перезапусти приложение — список обновится

## Ограничения

- **Не передаёт сигналы сам** — открывает встроенный Sub-GHz, который и занимается передачей. Это сделано осознанно: писать свой передатчик протоколов с нуля = 90% кода фирмварного субгерца. Зато надёжно работает с любыми форматами.
- Сортировка по имени, без вложенных папок (всё в одном плоском списке — это фича, а не баг: меньше кликов).

## Сборка

```bash
cd remote_hub
ufbt
```

Готовый FAP появится в `dist/remote_hub.fap`. Категория установки: `apps/Sub-GHz/`.
